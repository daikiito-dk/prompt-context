LEARNINGS.md — 蓄積ナレッジ

議事録全文ではなく、次に効く一行だけを残す。ファイルが太りすぎたら「今週有効なもの」だけ SESSION に転記する。

こんなときに使う

  • 同じ罠を別メンバーが踏むのを防ぎたい
  • モデルや外部 API の「クセ」をチームで共有したい
  • 長期プロジェクトで、昔の判断の経緯を手早く引きたい

運用のコツ

  • エントリは 時系列の逆順（新しいほど上） が読みやすい。
  • 「状況 → ひっかかり → 対処 → 再発防止」の4点セットを意識。
  • 推測と事実を分ける。事実だけを SESSION / SPEC に昇格させる。

テンプレート（Markdown）

# LEARNINGS — プロジェクト蓄積ナレッジ

> 長い議論の代替にはならない。**次の行動が変わる一行**だけを残す。

## アクティブ（直近・毎回のセッションに貼ってよいもの）
- （YYYY-MM-DD）**教訓**: … — **再発防止**: …

## ログ（時系列・新しい順）

### YYYY-MM-DD — 短い見出し
- **状況**:
- **何が起きたか**（事実）:
- **仮説・原因**（未検証は explicitly）:
- **取った対処**:
- **次に同じことを避けるルール**（1文）:
- **参照**（PR / チケット / ドキュメント）:

### YYYY-MM-DD — …

## アーカイブ（もう効かないが記録として残す）
- （YYYY-MM-DD）…

---
*LEARNINGS 最終整理: YYYY-MM-DD*
